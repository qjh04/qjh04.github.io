第一章 Stream流

创建不可变集合

既不可被修改的集合，不想让别人修改集合中的内容，可以用不可变集合。

如何创建

在List、Set、Map接口中都有一个of方法，此方法创建的集合不可修改，只能查询。

  • 当我们获取一个不可变的Set集合时，需保证参数唯一性，否则报错。
  • 创建Map集合时，键不能重复，否则报错。Map里面的of方法有上限，最多只能容纳20个参数，10个键值对。如果要超过10，可以用别的方法copyOf()或者ofEntries()。

Stream流

Stream流的思想

类似工厂流水线，每个工位都有其独特的功能。结合Lambda表达式，简化对数组集合的操作。步骤如下

  • 先得到一条Stream流
  • 利用各种api操作对象
    • 中间方法：过滤、转换。完成后还可以调用其他方法
    • 终结方法：输出、统计。完成后不可以调用其他方法
Stream流获取方式
获取方式 方法说明
单列集合 默认stream()方法
双列集合 没有方法，只能先转为entrySet或者keySet再用stream()方法
数组 Arrays静态方法stream()
零散数据 Stream接口中的静态方法of()，方法形参是一个可变参数，如果传递数组必须是引用数据类型
Stream流的中间方法

Stream流只能使用一次，所以用链式编程。修改流中的数据，不会影响源数据。

方法名称 说明
filter() 过滤
limit() 获取前几个元素
skip() 跳过前几个元素
distinct() 去重依赖hashcode和euqals方法
concat() Stream的静态方法，合并两个流为一个
map() 转换流中的数据类型

方法名称 说明

第二章 反射

反射

允许对成员变量、成员方法和构造方法的信息进行编程式访问。先获取，在解剖。

获取Class对象的三种方式

  • Class.forName(“全类名”)：源代码阶段
  • 类名.class：加载入内存
  • 对象.getClass()：创建对象后

利用反射获取构造方法

  • Constructor <? >[] getConstructors():返回所有公共构造方法对象的数组

  • Constructor <? >[] getDeclaredConstructors():返回所有构造方法对象的数组，包括私有

  • ConstructorgetConstructor(Class <? > … parameterTypes):返回单个公共构造方法对象
    ConstructorgetDeclaredConstructor(Class <? > … parameterTypes):返回单个构造方法对象

Constructor类中用于创建对象的方法

  • T newlnstance(Object. initargs):根据指定的构造方法创建对象
  • setAccessible(boolean flag):设置为true,表示取消访问检查

利用反射获取成员变量

  • Class类中用于获取成员变量的方法

  • Field[] getFields():返回所有公共成员变量对象的数组

  • Field[] getDeclaredFields():返回所有成员变量对象的数组

  • Field getField(String name):返回单个公共成员变量对象

  • Field getDeclaredField(String name):返回单个成员变量对象

Field类中用于创建对象的方法

  • void set(Object obj, Object value):赋值

  • Object get(Object obj)获取值。

利用反射获取成员方法

  • Method[] getMethods():返回所有公共成员方法对象的数组,包括继承的
  • Method[] getDeclaredMethods():返回所有成员方法对象的数组,不包括继承的
  • Method getMethod(String name,Class <? > … parameterTypes):返回单个公共成员方法对象
  • Method getDeclaredMethod(String name,Class <? > … parameterTypes):返回单个成员方法对象

Method类中用于创建对象的方法

  • Object invoke(Object obj, Object … args):运行方法
    • 参数一:用obj对象调用该方法
    • 参数二:调用方法的传递的参数(如果没有就不写)
    • 返回值:方法的返回值(如果没有就不写)

动态代理

无侵入式的给代码增加额外的功能，代理里面的方法就是需要被代理的对象的方法。可以通过接口保证，后面的代理和对象需要实现同一个接口，接口中的方法就是需要被代理的方法。

如何为Java对象创建一个代理对象?

java.lang.reflect.Proxy类:提供了为对象产生代理对象的方法:

  • public static Object newProxyInstance(ClassLoader loader, Class <? >[] interfaces, InvocationHandler h)

    • 参数一:用于指定用哪个类加载器,去加载生成的代理类，编写代码的类

    • 参数二:指定接口,这些接口用于指定生成的代理长什么,也就是有哪些方法

    • 参数三:用来指定生成的代理对象要干什么事情

在编译期间提供了一个监测机制，且减少了数据类型转换。泛型的本质是参数化类型，也就是所操作的数据类型被指定为一个参数。

泛型方法

在调用方法的时候独立指定泛型的具体类型。泛型方法可以被static修饰，但是泛型类中的普通成员方法不行。

定义语法

  • public与返回值中间非常重要,可以理解为声明此方法为泛型方法。
  • 只有声明了的方法才是泛型方法,泛型类中的使用了泛型的成员方法并不是泛型方法。
  • 表明该方法将使用泛型类型T,此时才可以在方法中使用泛型类型T。
  • 与泛型类的定义一样,此处T可以随便写为任意标识,常见的如T、E、K、V等形式的参数常用于表示泛型。

类型擦除

泛型信息只存在于代码编译阶段，进入JVM之前，会将与泛型相关的信息擦除，我们称之为类型擦除。

  • 无限制类型擦除：将所有泛型用Object类代替
  • 有限制类型擦除：将所有泛型用上限类型代替

桥接方法：保持接口和类的实现关系。接口会被擦除，实现类实现泛型接口时可能会直接定义泛型类型为某个类型不需要被擦除，这时就需要接口和实现类之间进行转换。可以用反射机制查看。

泛型数组

可以声明带泛型的数组应用，不能直接声明带泛型的数组对象。数组里面是集合。

反射常用的泛型类

反射中的构造方法可以加泛型，在利用反射机制创建对象时可以不用转换。
