面向对象

面向对象更注重对象的存在和对象的描述而面向过程更注重于本过程实现了什么功能。面向对象有三大特征，封装，继承，多态。
​封装，将对象的成员变量私有，开放相关的get和set方法让外部对象访问。还可以在相关方法里加上自己想加的业务需求。继承，继承父类的所有成员变量和成员方法，提高代码复用性，可以对需要重写的方法进行重写，为多态提供基础支持。多态，对于一个行为，不同的子类具有不同的表现形式。多态需要三个条件，继承父类，重写父类方法，父类引用指向子类对象，引用去调被重写的方法。

JDK、JRE和JVM的区别

JDK是java开发工具，JRE是Java程序运行环境，JVM是java虚拟机。三者的关系是，JDK包含JRE和Javac、Java等工具，JRE包含JVM和一些类库文件。将编译好的class文件可以直接放在windows或Linux系统的Java虚拟机上运行。

==和equals的区别

==判断的是引用地址是否一致，所以当判断两个String对象的时候，可能会出现字符一样，但是判断后不相等的情况，这时候需要用equals方法判断。没有重写的equals方法底层代码就是用==判断，String的是重写后的equals，会先判断引用是否相等，相等就返回true，不等会逐字符判断都相等也会返回相等。

final有什么作用？

可以修饰类、方法、变量。当修饰类时，此类不可以被继承。修饰方法时，方法不能被重写，但可以重载。修饰成员变量，成员变量可以在静态代码快赋值。修饰局部变量可以不赋值，但是后边赋值以后不能改变。修饰基本数据类型值不可以更改，修饰引用数据类型，不可以再指向另一个对象但是引用的值可以改变。

为什么局部内部类和匿名内部类只能访问局部final变量？

内部类和外部类级别一致，内部类不会因为定义在外部类方法里，方法执行完毕而销毁。因此，内部类在调用外部类的成员变量而外部类执行完毕销毁，就会产生问题。为了解决这个问题将外部类copy一份到内部类，又会出现数据不一致的问题，所以只能将要局部f变量用final修饰。

重载和重写的区别

重载发生在同一个类里，方法名必须相同，参数名、类型、个数、顺序可以不同，方法返回值、访问修饰符可以不同，如果只有类型修饰符不同会编译出错。
重写发生在一个子类继承父类的时候，子类重写父类拥有的方法。方法名、参数列表必须相同，返回值范围小于等于父类，抛出的异常小于等于父类，访问类型大于等于父类，private不允许重写。

接口和抽象类的区别

接口可以实现多个，抽象类只能继承一个。接口只能有public abstract()方法，抽象类可以有其他成员方法。接口只能有public static final类型的成员变量，抽象类可以有其他类型的成员变量。接口是对类的约束，抽象类是为了代码复用。接口定义了类的行为的有无不关心具体，抽象类更关注类的行为的实现。

List和Set的区别

List按进入顺序排序，可重复可以有多个Null对象，可以按iterator逐一遍历，也可以用get index方法。
Set无顺序，不能有重复，只允许有一个Null对象，推荐用iterator遍历。

hashCode和equals

hashCode获取对象的哈希值，本质上是int型数值，是确定该对象存储在哈希表上的索引位置。它的特点是能根据键快速找到对应的值。如果两个对象相等，hashCode一定相等，equals也返回true，如果两个对象hashCode相等，他们不一定相等。所以重写equals方法后还要重写hashCode方法。

ArrayList和LinkedList区别

ArrayList基于动态数组，底层连续存储，随机访问。因为数组长度固定，当数组存到一定数量需要扩容。扩容时可能用到头插法，头插法效率极低 因为要从后面一个一个拷贝往后移，而尾插法效率极高甚至比LinkedList还要好。
LinkedList基于链表，底层分散存储，适合增删改业务。不适合查询，因为要逐一遍历，遍历时用iterator不要用for循环，因为for循环每次用get方法取要遍历一次链表消耗极大，另外也不要使用indexOf方法，找不到目标值会遍历整个链表。

HashMap与Hashtable有什么区别

HashMap没有用synchronized修饰，线程非安全，Hashtable线程安全。HashMap允许Key和Value存储 Null值，Hashtable不允许。
底层实现原理，利用数组和链表。jdk8开始，当链表长度超过8，数组长度超过64，链表转为红黑树，元素以内部类Node节点存在。计算Key的hash值，二次hash对数组长度取余，得到对应数组下标，如果没有产生哈希冲突，直接将节点存入数组，如果产生哈希冲突，先进行equals比对，相同更新，不同添加到链表里，链表长度超过8并且数组超过64转换为红黑树，长度低于6红黑树在转回链表。如果Key为Null，存入0。
数组扩容问题，由于底层用数组实现，不能动态扩容，所以当长度不够用时，会重新new一个出来，再将原来的数组copy到新数组。

如何实现一个IOC容器

配置文件配置包扫描路径，定义一些注解，分别表示访问控制层、业务服务层、数据持久层、依赖注入注解、获取配置文件注解等。递归包扫描获取.class文件，添加到一个Set容器中。遍历这个Set容器，确定需要交给IOC管理的类，定义一个安全的Map容器管理这些类。遍历这个IOC容器对需要注入的类进行依赖注入。

什么是字节码

在机器和程序之间有一个接口，java虚拟机。编译程序只需要生成虚拟机能够理解的代码，然后解释程序将虚拟机代码转换为特定系统能够识别的机器码执行。供虚拟机理解的代码成为字节码。它只面向java虚拟机。

字节码的作用是什么

解决传统解释型语言执行效率低的问题，同时保留了解释型语言可移植性高的优点。所以java程序运行效率高，因为字节码只面向虚拟机，也解决了面对特定系统需要重新编译的问题。
